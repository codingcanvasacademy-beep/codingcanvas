# Concerns

### Technical Debt & Stability
1. **Authentication Email Limits:** Hard Supabase constraints rate-limited email token generation previously. The platform mitigated this using explicit Password structures. Constant awareness of quota checks is recommended when switching environments.
2. **Next.js Caching Strategies:** Substantial time has been lost to Next.js App Router implicitly cache-poisoning redirection flows post-authentication. Any future redirect operations **must** verify they aren't getting intercepted by old client-side storage.
3. **WebAssembly Heavy Lifting:** `blocks` loads Pyodide dynamically alongside Blockly. Although optimized with `dynamic(ssr: false)`, these payloads are inherently massive. We must be highly cautious with initial-load latency for devices running standard networks.

### Security
1. **Google Gemini LLM Proxies:** We pass unverified string values generated by end-users heavily into our `/api/ai` endpoints. While currently structured securely internally, prompt injection vulnerability stands out. Hard guardrails inside the server routes enforcing restricted JSON schemas rather than open-ended Markdown are critical to long term security. 
2. **Row Level Security (RLS) Edge-Cases:** Currently, standard mappings of `teacher_invites` and `free_class_requests` require rigid testing to ensure standard user-profiles cannot query outside their specific foreign-key mappings.
